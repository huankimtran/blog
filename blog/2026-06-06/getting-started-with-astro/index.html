Getting Started with Astro

Published: June 6, 2026
#astro #web #tutorial

Astro is a modern static site generator that lets you build fast, content-focused websites. Unlike traditional frameworks that ship heavy JavaScript bundles to the browser, Astro renders your pages to static HTML and only hydrates interactive components when needed.

Why Astro?

Astro’s philosophy is simple: ship less JavaScript. By default, every page is rendered to static HTML at build time. If you need interactivity, you can add it selectively using “islands” — isolated components that hydrate independently.

Key Features

  • Zero JS by default: Pages are pure HTML/CSS unless you opt into interactivity
  • Bring your own framework: Use React, Vue, Svelte, or just plain HTML
  • Content collections: Type-safe, validated content with Zod schemas
  • MDX support: Write content in Markdown with embedded components
  • Built-in integrations: RSS, sitemap, Tailwind CSS, and more

Setting Up Your First Project

# Create a new Astro project
npm create astro@latest my-blog -- --template minimal

# Navigate into the project
cd my-blog

# Start the dev server
npm run dev

Your site will be available at http://localhost:4321.

Content Collections

Astro’s content collections provide type-safe content management. Define a schema once, and every post is validated against it:

// src/content/config.ts
import { defineCollection, z } from "astro:content";

const blog = defineCollection({
  schema: z.object({
    title: z.string(),
    description: z.string(),
    pubDate: z.date(),
    tags: z.array(z.string()).default([]),
    draft: z.boolean().default(false),
  }),
});

export const collections = { blog };

Building for Production

When you’re ready to deploy:

npm run build

This generates a dist/ directory with static HTML, CSS, and minimal JavaScript. Deploy it anywhere — Netlify, Vercel, GitHub Pages, or your own server.
